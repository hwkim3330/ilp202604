How It Works

LiDAR 트래픽을 실시간 분석하여
TSN 스위치의 TAS 설정을 자동 생성하고 적용합니다
LiDAR 켜고 약 10초면 TAS 적용 완료
Pipeline

5-Step Automated Flow

서버가 LiDAR 패킷을 수신하는 즉시 타이밍 분석을 시작합니다. 별도 설정 없이 패킷 패턴에서 최적 TAS를 자동 도출합니다.

1
LiDAR Packet Reception
Ouster OS-1-16이 UDP 패킷을 전송합니다. LAN9662 스위치를 통과하여 서버에 도착합니다.
즉시
패킷 구조: 3,392 bytes = 16 columns × 212 bytes (LEGACY format)
전송 패턴: 32 packets/frame × 10 fps = 320 packets/sec
패킷 간격: 100,000 µs ÷ 32 = 3,125 µs
대역폭: 3,392 × 320 × 8 = 8.68 Mbps
2
Timestamp & Profiling
서버가 각 패킷의 도착 시간을 process.hrtime.bigint()로 기록합니다. 나노초 정밀도의 타임스탬프를 슬라이딩 윈도우에 축적합니다.
0.5초 ~ 10초
패킷 버퍼: 최근 500개 (약 1.56초 분량)
프레임 버퍼: 최근 100개 (약 10초 분량)
최소 데이터: 5프레임 (0.5초) 후 프로파일 생성 가능
안정 상태: 100프레임 (10초) 후 통계 수렴
3
Auto TAS Derivation
측정된 패킷 간격에서 TAS 사이클을 도출합니다. 와이어 타임에 마진을 곱해 TC7 슬롯을 계산합니다.
자동 (1초마다 갱신)
cycle = round(pkt_interval, 10µs) = round(3125, 10) = 3,130 µs
wire_time = (3392 + 20 + 14 + 4) × 8 / 1000 = 27.44 µs
TC7_slot = wire_time × 2.0 (margin) = 54.9 µs
BE_slot = cycle - TC7 - guards = 3130 - 54.9 - 2 = 3,073.1 µs
utilization = 54.9 / 3130 = 1.75%
4
WebSocket Streaming
프로파일과 auto-TAS를 1초 간격으로 브라우저에 스트리밍합니다. Timing 페이지에서 오실로스코프, 지터 히스토그램, TAS 오버레이로 시각화됩니다.
1초마다
WS 경로: /ws/lidar-timing-a
메시지 타입: pkt (매 패킷), profile (10프레임마다)
4개 패널: Packet Scope, Frame Detail, TAS Gate, Jitter Histogram
5
Push to LAN9662
"Push to Board" 버튼으로 TAS 설정을 스위치에 적용합니다. CoAP iPATCH로 YANG/CBOR 인코딩된 GCL을 전송합니다.
~2초
프로토콜: CoAP iPATCH (Ethernet) or Serial (fallback)
인코딩: YANG → SID → CBOR
대상: /ietf-interfaces:interfaces/interface[name='swp0']/ieee802-dot1q-sched-bridge:gate-parameters
도구: keti-tsn-cli
